Introduction

DocPortal AI is an enterprise-grade Retrieval-Augmented Generation (RAG) platform designed for intelligent document management, semantic search, AI-powered querying, and automated insight generation. The system combines hybrid search, vector databases, LLM orchestration, and asynchronous ingestion pipelines to enable scalable document intelligence across large collections of enterprise documents.

Architectural Note: Developed using a "Bring Your Own Key" (BYOK) model with free-tier APIs (Groq & Gemini) during development, while architected for seamless transition to enterprise-grade production APIs for deployment.

Core Features

Enterprise RAG Chat

AI-powered conversational interface that enables users to query documents using natural language with streaming responses and source-aware contextual retrieval.

Hybrid Search Engine

Combined BM25 keyword retrieval and vector similarity search using weighted ranking algorithms for improved semantic accuracy and relevance scoring.

Intelligent Ingestion

Automated ingestion pipeline capable of parsing, semantic chunking, and vector indexing for PDF, DOCX, TXT, and Markdown files.

Intelligence Briefs

AI-generated analytical briefs that aggregate insights across multiple documents using Gemini-based extraction pipelines and structured analytics generation.

System Architecture

DocPortalAI follows a layered enterprise microservices architecture designed for scalability, asynchronous processing, and modular AI orchestration.

DocPortalAI Enterprise RAG Architecture — Frontend, FastAPI Backend, Core Data Layer (PostgreSQL, Weaviate, S3), and AI & Cloud Providers (Groq, Nomic, Gemini).

1. Frontend Layer (React.js)

Built with React.js and Context API, the frontend provides the user interaction layer for document uploads, analytics dashboards, and real-time monitoring.

Major UI Components

  • Advanced Intelligence Dashboard: Interactive dashboard with sortable tables and document status monitoring.
  • Projects Dashboard: Central hub for managing all document projects — view project status, document counts, and jump directly into project-specific chat or analytics.
  • Project Insights Page: Per-project intelligence hub that surfaces AI-generated analytical briefs, document summaries, and key insights extracted across the entire project knowledge base.
  • Context-Aware RAG Chat UI: Conversational interface with streaming responses and source citations — supports both project-scoped and global cross-project querying.
  • Document Ingestion Portal: Drag-and-drop system with batch upload support and progress tracking.
  • Database Management Page: Multi-tenant collection browser with Weaviate vector store visibility, document count tracking, and collection-level controls.

Technical Workflows

1. Document Ingestion Pipeline
PDF Upload ──▶ Python PDF Parsing ──▶ Semantic Chunking ──▶ Nomic Embedding ──▶ Weaviate DB └──▶ Raw PDF Copy ──▶ S3 Storage

The PDF is parsed immediately on upload using Python libraries (PyMuPDF, python-docx). During parsing, a raw copy of the file is simultaneously persisted to S3 for archival. The parsed content is then semantically chunked, vectorized via Nomic Embeddings, and indexed into Weaviate for retrieval.

2. RAG Chat Pipeline
User Query ──▶ Hybrid Search ──▶ Result Reranking ──▶ Context Assembly ──▶ Groq LLM ──▶ Streamed Response

Top-ranked chunks are aggregated into an optimized context window, allowing Groq Llama 3.3 to generate contextual, source-backed responses streamed to the UI. The system supports two chat modes:

Project-Specific Chat

Scoped to a single project's document collection — ideal for deep-diving into a specific knowledge base with high-precision retrieval.

Security Architecture

Authentication

JWT-based auth with token expiration and refresh token support.

API Security

CORS whitelisting, rate limiting, and TLS 1.3 encryption for all pipes.

Data Privacy

PII scrubbing using local Ollama and encrypted S3 storage buckets.
